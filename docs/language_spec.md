## Общие сведения

Данный документ описывает лексическую структуру языка программирования **MiniCompiler** — упрощённого C-подобного языка, разрабатываемого в рамках учебного проекта по созданию компилятора.

Лексический анализ (сканирование) является первым этапом компиляции и отвечает за преобразование исходного текста программы в последовательность токенов — минимальных значимых единиц языка.

---

## Кодировка и набор символов

- **Кодировка:** UTF-8
- **Допустимые символы:**
  - Латинские буквы: `a-z`, `A-Z`
  - Цифры: `0-9`
  - Символ подчёркивания: `_`
  - Специальные символы, используемые в операторах и разделителях: `+ - * / % = ! < > & | ( ) { } [ ] ; , .`
  - Пробельные символы: пробел (` `), табуляция (`\t`), перевод строки (`\n`), возврат каретки (`\r`)
- **Недопустимые символы:** Любые другие символы (например, `@`, `#`, `$`, `^`, `` ` ``, `~` и т.д.) считаются ошибкой, если не находятся внутри строкового литерала или комментария.

---

## Лексическая грамматика (EBNF)

Для формального описания лексической структуры используется нотация **EBNF** (Extended Backus-Naur Form) со следующими обозначениями:

| Синтаксис | Значение |
|-----------|----------|
| `A \| B`  | Выбор: A или B |
| `{ A }`   | Повторение: ноль или более раз |
| `[ A ]`   | Опционально: ноль или один раз |
| `'a'`     | Терминальный символ |
| `(* ... *)` | Комментарий в EBNF |

## Ключевые слова

Ключевые слова — это зарезервированные идентификаторы, которые не могут использоваться в качестве имён переменных или функций.

**Список ключевых слов:**

```
if        else      while     for
int       float     bool      return
true      false     void      struct
fn
```

**Примечание:** Все ключевые слова являются **регистрозависимыми** и должны быть написаны строчными буквами.

---

## Идентификаторы

Идентификаторы используются для именования переменных, функций, структур и других пользовательских элементов.

**Правила формирования:**

```
letter     = 'a'..'z' | 'A'..'Z'
digit      = '0'..'9'
identifier = letter ( letter | digit | '_' )*
```

**Ограничения:**

- Должен начинаться с буквы (`a-z`, `A-Z`)
- Может содержать буквы, цифры и символ подчёркивания (`_`)
- Максимальная длина: **255 символов**
- Регистрозависимый: `counter`, `Counter` и `COUNTER` — разные идентификаторы
- Не может совпадать с ключевыми словами

**Примеры корректных идентификаторов:**

```
x
counter
_temp
value123
longIdentifierWithManyCharacters
```

**Примеры некорректных идентификаторов:**

```
123abc        (начинается с цифры)
my-var        (содержит дефис)
if            (совпадает с ключевым словом)
```

## Литералы

Литералы представляют собой фиксированные значения, записанные непосредственно в коде программы.

### Целочисленные литералы

**Формат:** последовательность десятичных цифр

```
int_literal = digit { digit }
```

**Диапазон значений:** от `-2³¹` до `2³¹-1` (от -2147483648 до 2147483647).  
Выход за пределы диапазона должен обнаруживаться на этапе семантического анализа (Спринт 3).

**Примеры:**

```
0
42
1024
2147483647
```

### Вещественные литералы

**Формат:** десятичное число с точкой (должны быть цифры как до, так и после точки)

```
float_literal = digit { digit } '.' digit { digit }
```

**Примеры:**

```
3.14
0.5
10.0
123.456
```

**Некорректные примеры:**

```
.5           (нет цифр до точки)
5.           (нет цифр после точки)
```

### Строковые литералы

**Формат:** последовательность символов, заключённая в двойные кавычки

```
string_literal = '"' { any_character_except_quote_or_newline } '"'
```

**Правила:**

- Не может содержать неэкранированные двойные кавычки внутри
- Не может содержать символы перевода строки (`\n`) (в текущей версии)
- Экранирование (`\n`, `\t`, `\"` и т.д.) **не поддерживается** в Спринте 1 (может быть добавлено позже)

**Примеры:**

```
"Hello, World!"
""
"file.txt"
"identifier with spaces"
```

**Ошибка:** незакрытая строка (отсутствует закрывающая кавычка до конца файла)

### Булевы литералы

Булевы литералы представлены ключевыми словами `true` и `false`, которые обрабатываются как отдельные токены.

**Формат:**

```
bool_literal = 'true' | 'false'
```

**Примечание:** Булевы литералы считаются ключевыми словами и не могут быть переопределены.

## Операторы

Операторы — это специальные символы или их комбинации, обозначающие действия над операндами.

### Арифметические операторы

| Оператор | Название          | Тип токена       |
|----------|-------------------|------------------|
| `+`      | Сложение          | PLUS             |
| `-`      | Вычитание         | MINUS            |
| `*`      | Умножение         | STAR             |
| `/`      | Деление           | SLASH            |
| `%`      | Остаток от деления| PERCENT          |

### Операторы сравнения

| Оператор | Название                | Тип токена |
|----------|-------------------------|------------|
| `==`     | Равно                   | EQ         |
| `!=`     | Не равно                | NE         |
| `<`      | Меньше                  | LT         |
| `<=`     | Меньше или равно        | LE         |
| `>`      | Больше                  | GT         |
| `>=`     | Больше или равно        | GE         |

### Логические операторы

| Оператор | Название                | Тип токена |
|----------|-------------------------|------------|
| `&&`     | Логическое И            | AND        |
| `\|\|`   | Логическое ИЛИ          | OR         |
| `!`      | Логическое НЕ           | NOT        |

### Операторы присваивания

| Оператор | Название                | Тип токена    |
|----------|-------------------------|---------------|
| `=`      | Присваивание            | ASSIGN        |
| `+=`     | Сложение с присваиванием| PLUS_ASSIGN   |
| `-=`     | Вычитание с присваиванием| MINUS_ASSIGN  |
| `*=`     | Умножение с присваиванием| STAR_ASSIGN   |
| `/=`     | Деление с присваиванием | SLASH_ASSIGN  |
| `%=`     | Остаток с присваиванием | PERCENT_ASSIGN|

**Примечание по разбору:** Многосимвольные операторы (`==`, `!=`, `<=`, `>=`, `&&`, `||`, `+=` и т.д.) должны распознаваться **до** односимвольных операторов, чтобы избежать ошибочной интерпретации.

## Разделители

Разделители (delimiters) используются для группировки выражений, разделения инструкций и аргументов.

| Символ | Название          | Тип токена |
|--------|-------------------|------------|
| `(`    | Открывающая круглая скобка | LPAREN  |
| `)`    | Закрывающая круглая скобка | RPAREN  |
| `{`    | Открывающая фигурная скобка | LBRACE  |
| `}`    | Закрывающая фигурная скобка | RBRACE  |
| `[`    | Открывающая квадратная скобка | LBRACKET |
| `]`    | Закрывающая квадратная скобка | RBRACKET |
| `;`    | Точка с запятой     | SEMICOLON |
| `,`    | Запятая             | COMMA     |
| `.`    | Точка (доступ к членам) | DOT    |

## Пробельные символы и комментарии

### Пробельные символы

Пробельные символы разделяют токены и игнорируются лексером (не порождают токенов).

**К пробельным символам относятся:**

- Пробел (`' '`)
- Табуляция (`'\t'`)
- Перевод строки (`'\n'`) — также увеличивает счётчик строк
- Возврат каретки (`'\r'`) — поддерживается для совместимости с Windows (`\r\n`)

### Однострочные комментарии

**Формат:**

```
single_line_comment = '//' { any_character } newline
```

Комментарий начинается с `//` и продолжается до конца текущей строки. Полностью игнорируется лексером.

**Пример:**

```c
// Это однострочный комментарий
int x = 42; // комментарий после кода
```

### Многострочные комментарии

**Формат:**

```
multi_line_comment = '/*' { any_character } '*/'
```

Комментарий начинается с `/*` и заканчивается на `*/`. Может занимать несколько строк. Полностью игнорируется лексером.

**Важно:** В текущей версии (Спринт 1) **вложенные комментарии не поддерживаются**. Строка `/* /* */ */` вызовет ошибку (закрывающий `*/` будет воспринят как конец первого комментария, а оставшийся текст вызовет синтаксическую ошибку).

**Ошибка:** незакрытый многострочный комментарий (достигнут конец файла без `*/`)

**Пример:**

```c
/* Это
   многострочный
   комментарий */
int y = 10;
```

## Полная спецификация в EBNF

Ниже представлена полная лексическая грамматика языка MiniCompiler в нотации EBNF:

```ebnf
(* Базовые элементы *)
letter      = 'a'..'z' | 'A'..'Z' ;
digit       = '0'..'9' ;
newline     = '\n' ;
carriage_return = '\r' ;
whitespace  = ' ' | '\t' | newline | carriage_return ;

(* Идентификаторы и ключевые слова *)
keyword     = 'if' | 'else' | 'while' | 'for' | 'int' | 'float' 
            | 'bool' | 'return' | 'true' | 'false' | 'void' 
            | 'struct' | 'fn' ;
identifier  = letter ( letter | digit | '_' )* ;

(* Литералы *)
int_literal     = digit { digit } ;
float_literal   = digit { digit } '.' digit { digit } ;
string_literal  = '"' (any_character_except_quote_or_newline)* '"' ;
bool_literal    = 'true' | 'false' ;

(* Операторы *)
operator    = arithmetic_op | relational_op | logical_op | assignment_op ;
arithmetic_op = '+' | '-' | '*' | '/' | '%' ;
relational_op = '==' | '!=' | '<' | '<=' | '>' | '>=' ;
logical_op    = '&&' | '||' | '!' ;
assignment_op = '=' | '+=' | '-=' | '*=' | '/=' | '%=' ;

(* Разделители *)
delimiter   = '(' | ')' | '{' | '}' | '[' | ']' | ';' | ',' | '.' ;

(* Комментарии *)
single_line_comment = '//' (any_character_except_newline)* newline ;
multi_line_comment  = '/*' any_character* '*/' ;
comment     = single_line_comment | multi_line_comment ;

(* Пробельные символы (игнорируются) *)
_whitespace = whitespace | comment ;

(* Токены *)
token       = keyword | identifier | int_literal | float_literal 
            | string_literal | bool_literal | operator | delimiter ;
```

## Примеры

### Пример 1: Простая программа

**Входной код:**
```
fn main() {
    int x = 42;
    return x;
}
```

**Ожидаемые токены:**
```
1:1 KW_FN "fn"
1:4 IDENTIFIER "main"
1:8 LPAREN "("
1:9 RPAREN ")"
1:10 LBRACE "{"
2:5 KW_INT "int"
2:9 IDENTIFIER "x"
2:11 ASSIGN "="
2:13 INT_LITERAL "42" 42
2:15 SEMICOLON ";"
3:5 KW_RETURN "return"
3:12 IDENTIFIER "x"
3:13 SEMICOLON ";"
4:1 RBRACE "}"
5:1 END_OF_FILE ""
```

### Пример 2: Выражения и операторы

**Входной код:**
```
if (a >= b && c != d) {
    result = a + b * c;
}
```

**Ожидаемые токены:**
```
1:1 KW_IF "if"
1:4 LPAREN "("
1:5 IDENTIFIER "a"
1:7 GE ">="
1:10 IDENTIFIER "b"
1:12 AND "&&"
1:15 IDENTIFIER "c"
1:17 NE "!="
1:20 IDENTIFIER "d"
1:21 RPAREN ")"
1:22 LBRACE "{"
2:5 IDENTIFIER "result"
2:12 ASSIGN "="
2:14 IDENTIFIER "a"
2:16 PLUS "+"
2:18 IDENTIFIER "b"
2:20 STAR "*"
2:22 IDENTIFIER "c"
2:23 SEMICOLON ";"
3:1 RBRACE "}"
4:1 END_OF_FILE ""
```

### Пример 3: Комментарии и строки

**Входной код:**
```
// This is a comment
name = "John Doe"; /* multi-line
comment */
```

**Ожидаемые токены:**
```
2:1 IDENTIFIER "name"
2:6 ASSIGN "="
2:8 STRING_LITERAL "\"John Doe\"" "John Doe"
2:19 SEMICOLON ";"
4:1 END_OF_FILE ""
```
